Lab 1: Data Flow, Context, and LLM-Powered Building

Import community workflows, build with context management, and generate workflows with an LLM

Skills practiced: Node-to-node data passing, expressions, context management, importing community workflows, using LLMs to generate and iterate on workflows

Prerequisites: n8n running locally, API keys configured (Gemini or Grok), Lesson 1 talk completed

The Scenario

Your team just got access to n8n and everyone's excited, but nobody has real experience building workflows yet. Your manager saw a demo and now wants three things by end of day: a working data pipeline that pulls information from the internet and transforms it, proof that you can grab and customize community workflows instead of building everything from scratch, and a workflow that was generated by an LLM so the team can evaluate whether that's a viable approach going forward. Three exercises, three deliverables.

1

Import, Explore, and Extend a Community Workflow

The Challenge

Go to the n8n workflow template library at n8n.io/workflows/1700 and import the "Very Quick Quickstart" template into your local n8n instance. This workflow demonstrates the basics of data flow between nodes using a training data store and HTTP requests.

Once you've imported it and verified it runs, your job is to extend it by adding two new nodes that transform the data before it reaches the final output. The specific transformation is up to you, but the extended workflow must modify the data in a visible way (filter it, reshape it, enrich it, or add computed fields).

Then, browse n8n.io/workflows and find one additional workflow template that interests you. Import it, run it, and be ready to explain to a neighbor what it does and how data flows through it.

Acceptance Criteria

  • The "Very Quick Quickstart" template (workflow #1700) is imported and runs successfully
  • You've added at least 2 new nodes that transform the output data in a visible way
  • You can click on any node and explain what data it receives and what data it outputs
  • You've imported one additional community workflow and can explain its data flow
  • You can export your modified workflow as JSON (this skill matters for Exercise 3)

Hints

Hint 1: How to import a template

On the n8n.io/workflows page, click "Use workflow" on any template. You can copy the JSON and paste it into n8n via Import from URL or Import from file in the n8n menu (the three dots or hamburger menu in the top-right of the canvas).

Hint 2: What nodes to add for transformation

Good candidates: Edit Fields (rename or reshape data), IF (filter based on a condition), Code (write a JavaScript snippet to compute new fields), or Sort (reorder items). Pick whichever feels most useful for the data you're working with.

Hint 3: Finding a good second template

Try browsing by category on n8n.io/workflows. Some good beginner-friendly ones:

  • Workflow #159 "Send RSS Feed Data to Webhook" — shows filtering and routing
  • Workflow #4709 "Daily News Digest" — shows schedule triggers and merging
  • Or search for a tool you already use (Slack, Gmail, Google Sheets)
Step-by-step guide (optional)
  1. Open your browser and go to https://n8n.io/workflows/1700-very-quick-quickstart/
  2. Click "Use workflow" and copy the workflow JSON
  3. In your local n8n (http://localhost:5678), create a new workflow
  4. Click the three-dot menu → "Import from clipboard" and paste the JSON
  5. Click "Test workflow" to verify it runs. Inspect the output of each node.
  6. Add an Edit Fields node after the last data node. Configure it to rename one field or add a new computed field (e.g., add a field called processed set to true)
  7. Add a Filter or IF node to filter out some items based on a condition
  8. Run the workflow again and verify your new nodes appear in the execution
  9. Go back to n8n.io/workflows, browse templates, pick one that looks interesting, import it
  10. Export your modified quickstart workflow: three-dot menu → "Download" to save the JSON file
2

Build a Context-Aware Multi-Step Workflow

The Challenge

Build a workflow from scratch that demonstrates proper context management across at least 4 nodes. The workflow should:

  1. Start with a trigger (manual trigger is fine)
  2. Fetch data from an external source (an API call, RSS feed, or even a static JSON input)
  3. Pass that data through at least two processing steps where each step needs context from a previous step (not just the immediately preceding node)
  4. Produce a final output that combines information from multiple earlier nodes

The key constraint: at least one node must reference data from a node that is not its direct predecessor. This forces you to deal with n8n's data scoping.

Acceptance Criteria

  • Workflow has at least 4 nodes (trigger + fetch + 2 processing + output)
  • At least one node uses an expression {{ }} that references data from a non-adjacent previous node
  • The final output combines information from at least two different earlier nodes
  • You can explain why the data scoping required you to handle context explicitly

Hints

Hint 1: A concrete scenario to build

Try this: Manual Trigger → HTTP Request (fetch a joke from https://official-joke-api.appspot.com/random_joke) → Edit Fields (extract setup and punchline plus add a timestamp) → Code node (combine the joke with the timestamp into a formatted message) → final Set node that produces a summary referencing the original API response AND the formatted message.

Hint 2: How to reference non-adjacent nodes

In n8n expressions, you can reference any previous node by name using {{ $('NodeName').item.json.fieldName }}. This is how you pull data from a node that isn't your direct input.

Hint 3: The Merge node pattern

If you're struggling with context, the Merge node is your friend. It takes two inputs and combines them. You can use it to bring together data from two separate branches or to merge current data with earlier data.

Step-by-step guide (optional)
  1. Create a new workflow. Add a Manual Trigger node.
  2. Add an HTTP Request node. Set it to GET https://official-joke-api.appspot.com/random_joke. Connect it to the trigger. Test it.
  3. Add an Edit Fields node. Map setup and punchline from the HTTP response, and add a new field fetchedAt set to the expression {{ $now.toISO() }}.
  4. Add a Code node. Reference the original HTTP response using $('HTTP Request').item.json to get the joke type, and combine it with the current input to produce a formattedMessage.
  5. Add a final Edit Fields node that produces three fields: originalJokeId (from $('HTTP Request').item.json.id), formattedMessage (from the Code node), and processingSteps set to "fetched → extracted → formatted → summarized".
  6. Run the workflow end to end. Click on each node and trace the data flow.
3

Generate a Workflow with an LLM

The Challenge

Use an LLM (ChatGPT, Claude, Gemini, or whichever you prefer) to generate a complete n8n workflow as JSON, then import it into your local n8n instance, test it, find at least one issue, and prompt the LLM to fix it.

The workflow you ask the LLM to generate should do something useful and contain at least 3 nodes. It must be different from what you built in Exercises 1 and 2.

The goal isn't to get a perfect workflow on the first try. The goal is to practice the generation loop: describe → generate → import → test → identify issues → prompt for fix → re-import.

Acceptance Criteria

  • You prompted an LLM and received a valid n8n workflow JSON in response
  • The generated workflow imports successfully into n8n (even if it doesn't run perfectly yet)
  • You identified at least one issue with the generated workflow
  • You prompted the LLM with the specific issue and received an updated JSON
  • The final version of the workflow runs (or at least progresses further than the first attempt)
  • You saved the final workflow JSON file

Hints

Hint 1: How to prompt for workflow generation

Be specific. A good prompt includes:

  • What the workflow should do (step by step)
  • What trigger type to use
  • What node types you want (HTTP Request, Code, Edit Fields, IF, etc.)
  • That you want the output as valid n8n workflow JSON that can be imported directly

Example: "Generate an n8n workflow JSON that: starts with a manual trigger, makes an HTTP GET request to [some API], filters the results to only include items where [condition], and outputs the filtered results to a Set node with specific fields. Return only the JSON, no explanation."

Hint 2: Common issues with LLM-generated workflows

The most common problems:

  • Wrong node type names — the LLM might use a node type that doesn't exist in your n8n version
  • Broken expressions — the {{ }} syntax might be slightly off
  • Missing credential references — nodes that need API keys won't have them configured
  • Incorrect JSON structure — the workflow JSON might not match n8n's expected schema

When you find an issue, copy the error message and the relevant JSON into your next prompt.

Hint 3: A good workflow to generate

If you're not sure what to ask for, try: "Generate an n8n workflow that takes a manual trigger, fetches the top 5 Hacker News stories from the API (https://hacker-news.firebaseio.com/v0/topstories.json), then for each story ID fetches the story details, and collects the titles and URLs into a final summary."

Step-by-step guide (optional)
  1. Open your preferred LLM (browser-based or local)
  2. Write a prompt describing the workflow you want. Include: trigger type, node types, what each step does, and explicitly ask for "valid n8n workflow JSON that I can import directly."
  3. Copy the JSON response
  4. In n8n, create a new workflow → three-dot menu → "Import from clipboard"
  5. If the import fails, paste the error back to the LLM and ask it to fix the JSON structure
  6. If the import succeeds, click "Test workflow" and see what happens
  7. Inspect each node's output. Find something that's wrong or not working as expected.
  8. Go back to the LLM. Paste the specific issue: "Node X fails with error Y" or "The expression in node X returns undefined." Ask for a targeted fix.
  9. Copy the updated JSON, re-import, and test again.
  10. Save the final working (or improved) JSON file for reference.

Wrap-Up

You've now done the three things that make someone productive in n8n fast: you imported and customized a community workflow (so you never build common patterns from scratch), you built a workflow from scratch with proper context management across nodes (the skill that separates working workflows from broken ones), and you used an LLM to generate and iterate on a workflow (which is going to be how most workflows start in the near future). All three of these workflows are yours to keep, reference, and build on in Lesson 2.
