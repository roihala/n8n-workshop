Build a Superpowered AI Agent

Teams of 2-3. Connect to real data. Do real thinking. Make people go "wait, it can do that?"

PrerequisitesLessons 1 and 2 completed (talks and labs). n8n running locally. API keys configured (Gemini or Grok).
Team sizePairs or groups of three
1

What Should Your Agent Do?

We're not building a demo. We're not automating a boring process. You're going to build an AI agent that does something that makes people go "wait, it can do that?" Something that connects to real data, thinks about it, and produces an output that would take a human an hour to do manually. By the end of this session, you'll have a working agent you can show to your team tomorrow.

The Discovery Exercise

As a team, answer these questions. Write the answers down.

1. What would blow your mind if an AI could just... do it?

Think big. Think about things that require reading, understanding, judgment, synthesis.

  • Connect to your Google Drive, read through 20 accounting documents, and produce a one-page executive summary with the three things your CFO needs to know
  • Scrape today's top news, cross-reference with your company's industry, and produce a morning brief that says "here's what matters to you today and here's what you should do about it"
  • Read a GitHub repository's last 50 commits and generate a changelog, a risk assessment, and a list of things that need code review
  • Take a job posting URL, research the company, analyze the role, and produce a personalized cover letter plus interview prep notes
  • Monitor a Telegram channel, detect when someone asks a question, research the answer from your docs, and respond automatically
  • Read a CSV of customer feedback, identify patterns, rank issues by severity, and draft a product roadmap recommendation

2. What real data can you connect it to?

3. What should the output look like?

  • A structured report delivered to Slack or email
  • A Google Doc created automatically with analysis and recommendations
  • A spreadsheet populated with extracted and organized data
  • A Telegram message with a daily intelligence briefing
  • A dashboard-ready JSON with scored and categorized insights

Write Your Agent Pitch

"Our agent connects to [data source], [does something intelligent with it], and delivers [impressive output] to [destination]."

Examples:

  • "Our agent connects to Google Drive, reads all invoices from this month, extracts vendor names, amounts, and due dates, identifies anything overdue, and posts a financial summary to Slack with action items."
  • "Our agent scrapes 5 news sources every morning, filters for articles relevant to our industry, analyzes sentiment and potential business impact, and sends a one-page morning brief to the team's email."
  • "Our agent reads a GitHub repo URL, analyzes the code structure, identifies the tech stack, estimates complexity, and produces a technical assessment document in Google Docs."
  • "Our agent takes a product description, researches competitor products on the web, compares features and pricing, and generates a competitive analysis spreadsheet."

Design the Agent

Step 1: Draw the Flow

Map what the agent does from trigger to output. Mark where the AI thinks.

Example: Morning News Intelligence Agent

Schedule Trigger (7:00 AM)
    → HTTP Request (fetch RSS feed 1)
    → HTTP Request (fetch RSS feed 2)
    → Merge (combine all articles)
    → AI Node: "Filter these articles. Which ones are relevant to [industry]?
               Return JSON: [{title, url, relevance_score, why_it_matters}]"
    → Code Node (parse, sort by relevance)
    → AI Node: "Given these relevant articles, write a 3-paragraph morning brief.
               What happened, what it means for us, what we should do."
    → Gmail: Send to team@company.com
Example: Document Intelligence Agent

Manual Trigger (or Form Trigger with a Google Drive folder URL)
    → Google Drive: List files in folder
    → Loop: For each file → Google Drive: Download → AI Node: Extract key information
    → Merge all extractions
    → AI Node: "Synthesize all extracted information into an executive summary.
               Identify the 3 most important findings and any red flags."
    → Google Docs: Create new document with the summary
    → Slack: Post link to the document in #reports

Step 3: Write System Prompts

For every AI node, write the prompt before you build. This is what makes the agent smart.

  • Role: one sentence ("You are a financial analyst reviewing invoices")
  • Input: what data it receives
  • Output format: be extremely specific, especially if you need JSON
  • Constraints: what it should NOT do ("Don't make up information not present in the documents")

Build It

Open n8n at http://localhost:5678. Create a new workflow.

Build Order

1. Skeleton first.

Lay down all nodes and connect them. Placeholder prompts for AI nodes. Sample data after the trigger. Run it. Data flows end-to-end without errors.

2. Connect real data.

Set up credentials for your data sources (Google Drive, Gmail, Sheets, etc.). Test the connection. Get real data flowing. This is the first "wow."

3. Make the AI think.

Replace placeholder prompts with your real system prompts. Test each AI node individually with "Test step." This is the second "wow."

If you're getting structured output (JSON from AI), common fixes:

  • Add "Return ONLY valid JSON, no markdown, no explanation" to your system prompt
  • Add a Code node: text.replace(/```json\n?/g, '').replace(/```\n?/g, '')
  • Use the Structured Output Parser node instead of manual parsing

4. Wire decision logic.

Add IF/Switch nodes if your agent needs to branch. Connect parsed AI output to drive routing.

5. Deliver the output.

Connect the final output to a real destination. Send the email. Post to Slack. Create the Google Doc. This is the third "wow."

6. Test with real data.

Run the full workflow end-to-end with real input. Not sample data. Real documents, real emails, real URLs.

Debugging

ProblemFix
"No data" at a nodeClick previous node, check output, fix field mapping
AI returns garbageCheck Input tab. Fix system prompt. Be more specific.
IF/Switch always goes one wayTest expression in Expression Editor with actual data
Credential errorRe-authenticate in Settings → Credentials
Google Drive / Sheets permission errorRe-authorize OAuth. Make sure the right scopes are enabled.

When You're Stuck

  1. Inspect the data. Click any node after execution. 90% of bugs are data shape issues.
  2. Test nodes individually. "Test step" runs just that node.
  3. Simplify. Replace the complex part with something that works, then add complexity back.
  4. Search n8n.io/workflows for a template doing something similar.
  5. Raise your hand.

Polish

  • Name your nodes descriptively ("Analyze Financial Docs" not "AI Node 3")
  • Add sticky notes to label sections of the canvas
  • Add error handling on critical nodes (Continue on Error)
  • Export the workflow JSON as a backup

Demo

Each team demos their agent live.

Demo Format

  1. The pitch. One sentence. What does your agent do?
  2. Trigger it. Run it live. Show real data going in.
  3. Walk the AI. Point to where the agent is thinking. Show what it decided and why.
  4. Show the result. Open Slack and show the message. Open Google Docs and show the report. The output should speak for itself.
  5. What's next. How would you make this even better?

What We're Looking For

  • The "wait, it can do that?" reaction. Did the output genuinely impress?
  • Real data in, real output out. Not sample data. Not hardcoded text.
  • AI that thinks. The agent analyzed, synthesized, or judged something.

Reference: n8n Patterns

The Classifier-Router

Input → AI Classify (JSON: {category, priority}) → Parse → Switch [category]
    ├── TYPE_A → handler_a ──┐
    ├── TYPE_B → handler_b ──┼── Merge → output
    └── TYPE_C → handler_c ──┘

n8n nodes: Basic LLM Chain + Code Node (parse) + Switch Node.

The Generator-Critic Loop

Generate (AI Node) → Evaluate (AI Node, JSON: {score, feedback})
    → IF [score >= threshold]
        ├── YES → output
        └── NO  → Generate (loop back with feedback, max 3 iterations)

n8n nodes: Two Basic LLM Chain nodes + Code Node + IF nodes.

The Supervisor-Delegates

Input → Manager Agent [which specialist?]
    ├── Specialist A (Agent Tool)
    ├── Specialist B (Agent Tool)
    └── Specialist C (Agent Tool)
→ output

n8n nodes: AI Agent (manager) + AI Agent nodes wrapped as Agent Tools.

The Human-in-the-Loop Gate

AI generates recommendation → Send for review (Slack/email)
    → Wait Node → Webhook receives response
    → IF [approved] → execute action

n8n nodes: Basic LLM Chain + Slack/Gmail + Wait Node + Webhook + IF.
